# VetVault — AGENTS.md

## Repo structure

Monorepo with npm workspaces (`apps/*`, `services/*`, `packages/*`). No turborepo/nx/lerna. Each workspace has its own `package.json`, `tsconfig.json`, `.env`, and `node_modules`.

| Path | Tech | Entry |
|---|---|---|
| `services/api-backend/` | Fastify 5 + Drizzle ORM + PostgreSQL | `src/server.ts` (runs via `tsx`) |
| `apps/web-app/` | React 19 + Vite 8 + React Router 7 | `src/main.tsx` |
| `apps/mobile-app/` | React Native — **stub, no code yet** |
| `packages/shared/` | Shared TS types + utils | `src/index.ts`, build output `dist/` |

## Commands

Run all commands from each workspace's own directory. No root-level scripts.

### API backend (`services/api-backend/`)

```bash
npm run dev              # tsx watch src/server.ts on :5000
npm run db:setup         # reset + push + seed + import-vets + import-productos
npm run db:seed-mock     # optional fake data for local dev
npm run db:push          # sync Drizzle schema to Postgres
npm run db:studio        # Drizzle Kit Studio
```

### Web app (`apps/web-app/`)

```bash
npm run dev              # Vite dev server on :5173
npm run build            # tsc -b && vite build
npm run lint             # ESLint (ts,tsx only)
```

### Shared package (`packages/shared/`)

```bash
npm run build            # tsc -> dist/
```

**Must build `@vetvault/shared` before building `web-app`** — web-app depends on `"@vetvault/shared": "*"` resolved from workspace.

## Key conventions

- **Module systems differ**: `api-backend` is CommonJS (`"type": "commonjs"`), `web-app` and `shared` are ESM.
- **API base** is hardcoded to `http://localhost:5000/api` in `apps/web-app/src/api/client.ts`.
- **Auth**: JWT in HttpOnly cookie named `token`. Endpoints use `verifyToken` preHandler. Session restored by calling `GET /api/usuarios/me` on mount.
- **All API routes** are prefixed with `/api` and registered in `services/api-backend/src/routes/index.ts`. Swagger docs at `/documentation`.
- **No tests** in any package — no test script, runner, or fixtures exist.
- **No formatter/Prettier config** — ESLint only in web-app.
- **DB migrations** live in `services/api-backend/drizzle/` generated by `drizzle-kit generate`.
- **Catalog data** (SENASA products, Córdoba veterinarians registry) imported via `npm run db:import-*` scripts from local JSON files at `services/api-backend/`.
- **Mobile app** is a placeholder with only a README.

## Session — Jun 24, 2026

### Goal
Implementar Fase 1 (Agenda + Vacunación) y Fase 2 (Pacientes + Tratamientos) del agente IA para veterinarios, con diferenciación de límite de function calls por rol.

### Constraints & Preferences
- Las consultas a BD deben ir en los servicios, no en tools.ts.
- Los veterinarios deben tener más function calls que los propietarios (8 vs 4).
- El agente debe inferir fechas relativas ("mañana", "hoy") automáticamente.
- Las sugerencias del chat deben mostrar 3 aleatorias de un pool más grande.
- Todo en español.
